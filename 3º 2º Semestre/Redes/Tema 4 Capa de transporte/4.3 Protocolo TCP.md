- Características:
	- Fases de transmisión:
		- 1. Establece conexión
		- 2. Transferencia de datos
		- 3. Cierre de conexión
	- Mecanismos de control de errores:
		- Numeración de segmentos
			- Cada segmento lleva un número de secuencia
			- Indica la posición que tiene el primer byte de datos del segmento del mensaje
		- Confirmaciones del receptor
			- Cuando le llega un segmento el receptor confirma la llegada
		- Retransmisión de segmentos:
			- Si no llega lo reenvía
	- ![[Pasted image 20260409093457.png]]
	- Flags:
		- URG: Puerto Urgente
		- ACK: Número de confirmación válido
		- PSH: Si los datos se deben pasar inmediatamente 
		- RST: Abortar conexión
		- SYN: Para establecer conexión
		- FIN: Finalizar conexión
- Control de errores:
	- ![[Pasted image 20260409094028.png]]
	- Los campos que buscamos en esta ventana de tiempo son los siguientes:
		- Seq (la secuencia)
		- Bits que participan:
			- ACK
			- SYN
			- FIN
			- RST
	- El receptor en el ejemplo no está mandando datos, solo es el emisor
	- Formato cajas:
		- SEQ NAck FIN SYN ACK RST
		- NAck va de la mano con ACK por lo que si NAck = - ACK = 0
		- Si no mandamos datos mandamos siempre el primer byte de información (x o y)

- Control de errrores:
	- Hay algunas simplificaciones
		- Solo se transmiten datos en un sentido (Emisor -> Receptor)
			- Esto hace que los campos del emisor sean solo el campo de SEQ mientras que el receptor solo tiene el NºACK 
		- Se supone que todos los segmentos transportan 100 bytes de datos
		- Se supone que el número de secuencia inicial del Emisor es SEQ 1
	- Ejemplo 1: Retardo en la transmisión de un segmento
		- 1. El receptor recibe segmentos fuera de secuencia
		- 2. El emisor recibe un ACK fuera de secuencia
		- 3. El receptor recibe el segmento faltante
		- ![[Pasted image 20260410114641.png]]
	- Ejemplo 2: Pérdida de un segmento 
		- 1. El receptor recibe segmentos fuera de secuencia
		- 2. El emisor recibe tres ACKs duplicados con el mismo identificador
		- 3. El receptor recibe el segmento faltante
		- ![[Pasted image 20260410115238.png]]
	- Ejemplo 3: Temporizador de retransmisión
		- Se usa para comprobar la posibilidad de que el emisor no reciba confirmaciones
		- Por cada nuevo segmento transmitido se inicia un temporizador de retransmisión
		- El segmento se retransmite si ha expirado el temporizador
		- CASOS:
			- 1: Se pierde una confirmación per llega la siguiente confirmación antes de que expire el temporizador

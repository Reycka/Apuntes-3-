- UDP (User Datagram Protocol)
	- **No garantiza** un servicio externo
	- Se usa en programas donde la velocidad es más importante que la fiabilidad
- TCP (Transmission Control Protocol)
	- Protocolo de transporte orientado a conexión
	- Garantiza un servicio extremo a extremo fiable
		- Detecta segmentos perdidios o erróneos de datos
		- Detecta segmentos duplicados y los descarta
		- Ordena los segmentos en el destino y los entrega de forma ordenanda
		- Se usa en programas donde lo más importante es la fiabilidad

- Modelo Cliente servidor
	- Patrón de comunicación más usado en internet
	- Cliente:
		- Aplicación o proceso que inicia la conexión o el intercambio de datos con la máquina remota (servidor)
		- La aplicación cliente normalmente la arranca un usuario cuando quiere utilizar un servicio de la red
	- Servidor:
		- Aplicación o proceso que recibe y acepta la solicitud de conexión o intercambio de datos del cliente
		- Esta aplicación normalmente está ejecutándose continuamente en la máquina remota esperando solicitudes de cliente
- Puertos:
	- Cada proceso o aplicación de red se identifica por un número de puerto
	- **Puerto en cliente**
		- Cuando se arranca este proceso el SO del cliente asigna un número de puerto libre, >1023 ya que los anteriores son para superusuario
	- **Puerto en servidor** 
		- El cliente conoce de antemano este puerto, ya que tiene un número de puerto fijo

- Sockets:
	- Al establecerse un canal de comunicación los extremos de dicho canal se denominan sockets
		- Un socket se identifica de forma única con 3 parámetros:
			- Dir IP
			- nº puerto
			- Protocolo
		- Esto permite el transporte entre ambas partes que unen el socket

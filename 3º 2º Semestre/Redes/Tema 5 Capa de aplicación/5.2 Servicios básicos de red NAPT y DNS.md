- NAPT
	- Características:
		- Servicio de red que permite la traducción de direcciones IP privadas en direcciones IP públicas y viceversa
		- Su objetivo es que varios dispositivos de una red privada puedan acceder a internet con una única IP pública
	- Funcionamiento:
		- El router cambia la IP privada por la pública
		- Modifica el puerto de origen para identifcar las conexiones
		- Mantiene una tabla de asignación para recordar qué puerto corresponde a qué dispositivo: Esta tabla funciona de esta manera:
			- 

| IP privada                        | IP pública                                    |
| --------------------------------- | --------------------------------------------- |
| IP priv A : port 9302 (aleatorio) | IP pública del router : port 2948 (aleatorio) |
Asocia el puerto de la IP privada y el de la IP pública de forma que cuando le llegue un mensaje del con el puerto de la pública sepa a que privada tiene que mandarle

- Cuando la respuesta llega desde internet, el router revisa la tabla y reenvía el paquete a la IP privada

- DNS
	- Cuando queremos comunicarnos con otra máquina todo empieza por conocer su IP, este protocolo busca sustituir la IP por un nombre, es decir, asociar un nombre real a dicha IP
	- Un servidor es un Software, NO UNA MÁQUINA, ya que una máquina puede tener varios servidores
	- Problemas --> Soluciones:
		- Colisisiones de nombres --> Espacio de dominios de nombres:
			- Asociar a cada recurso un nombre único, en formato alfabético y con jerarquía
		- Escalabilidad --> Mapeo entre nombre - IP
			- Asociar una dirección IP y un nombre de forma unívoca
			- Base de datos Distribuida Jerárquica (estructura de árbol) Escalable Información almacenada en registros RR (Resource Records)
	- ¿Qué es un servidor DNS?
		- Conjunto de programas y protocolos que utiliza una arquitectura cliente servidor
		- Se encarga de traducir nombres simbólicos a direcciones IP. Asociar un nombre a ia IP se llama Resolución DNS. 
			- Este resolver DNS puede estar en el propio router o que seea la propia compañía
	- Elementos DNS:
		- Espacio de dominios de nombres: Jerarquía de árbol donde cada nodo tiene informacion del dominio
		- ![[Pasted image 20260430092318.png]]
		- ![[Pasted image 20260430092443.png]]
		- Servidores de nombres: Mantienen y proporcionan información del espacio de dominios
		- Resolvers (Resolutor DNS): Se encargan de generar las consultas, obtener la información solicitada y ofrecérsela al usuario
		- Cliente DNS: plicación de usuario que realiza las consultas DNS al servidor (ej. navegador web
	- Zonas de autoridad:
		- ![[Pasted image 20260430092616.png]]
		- La zona de raíz es gestionada por el NIC, solo conocen las IPs de todos los servidores DNS de los dominios de nivel superior
		- Zonas de nivel superior:
			- Cada dominio del nivel superior es una zona de autoridad distinta
			- Están gestionadas por el NIIC
			- Cada zona tiene un número variable de DNS
			- Conocen las IPs de todos los servidores DNS de sus subdominios
			- NO CONOCEN la IP del resto de máquinas del resto de subdominios
			- ![[Pasted image 20260430093053.png]]
		- Zonas de nivel Subdominio
			- Conoce a las submáquinas (Nombres e IPs) asi como el contexto del root al que pertenecen
		- Tipos de servidores:
			- Servidores autoritativos:
				- Tiene la información del dominio y responden la consulta que reciben del resolver
				- Dos tipos:
					- Servidores primarios o maestros
						- Mantiene la base de datos con la información sobre la zona
					- Servidores secundarios o esclavos
						- Copia del primario
			- Servidores de sólo cacheo
				- No mantiene zonas
				- Almacenan de forma temporal los RRs de distintos dominios que han obtenido consultando a los servidores autoritativos las peticiones que recibe de los clientes para utilizarlas en caso de una nueva consulta
				- Se usan como Resolvers
				- Funcionamiento:
					- Consultas/respuestas DNS
						- Iterativas
						- Recursivas
					- Transferencia de zona
						- Mecanismo de replicación de ficheros de la zona
					- Actualizaciones dinámicas
						- Mecanismo utilizado para actualizar los ficheros de zona de un servidor DNS
					- Notificaciones
						- Transacciones que usa un servidor maestro para notificar cambios en su base de datos

- Punto a punto, todos los ordenadores se relacionan entre otros con un cable que se une de un ordenador a todos los demás se usa en redes privadas que no se van a ampliar
	- Pegas:
		- Si se agrega uno nuevo es imposible
		- No hay un punto global que permita relacionar con todos a la vez
	- Beneficios:
		- Es muy rápido
		- Te desentiendes de donde conecta cada uno
-  Medio compartido, donde hay una estructura hardware ya sea un Bus o un HUB que se conecta a todos los dispositivos y estos se comunican gracias a que todos están conectados con el Bus o el Hub
	- Pegas:
		- Aquí no hay privacidad,  todos ven la información ya que están 
		- Puede haber colisiones (2 personas mandan mensajes a la vez) entonces necesitamos que los ordenadores tengan un gestor de colisiones
		- Aquí hay que comunicar un identificador para saber a quién hay que mandarlo y quien lo ha mandado, ya que los ordenadores tienen todos distinto ID flow carta, para ello se le asignan campos al mensaje (requisitos para mandar el mensaje)
	- Beneficios:
		- Permite traer gente nueva
		- Se pueden relacionar todos con todos
		
- Switches: estos aislan la información como en el punto a punto de los dispositivos conectados al mismo, este permite a su vez mandar un mensaje a todos. Este empieza sin saber nada y según avancen los mensajes le va llegando información de donde es, quien es y eso el switch lo almacena en una tabla para identificar la posición de cada usuario, en caso de tener varios mensajes conjuntos el switch tiene una cola donde se guardan las cosas y esta se va vaciando una a una
	
- IDENTIFICADORES IPS
	- Nombre del ordenador
	- El responsable de asignarle la IP al ordenador es el administrador red, esto lo haremos en el Lab con máquinas virtuales
	- La tarjeta de red es la encargada de sacar y tener el identificador / IP, sin ella no se puede hacer nada 
	- Si no se asigna a nadie el mensaje se hace un broadCast que hace la llamada a TODOS los usuarios de la red

- DIRECCIÓN MAC:
	- Es el sello del fabricante, es el DNI del ordenandor
	- Los MAC referenciados son el enlace para ir al destino, Si ambas partes coinciden entonces estas 


- Resumen:
	- Las IPS son del fuente y el destino mientras que la MAC son los enlaces a los que tengo que llegar allí si quiero ir de A --> C y las relaciones son A --> B --> C siendo B mi router, A mandará el mensaje con estas características: IP fuente A IP destino C MAC fuente A MAC destino B y luego de llegar a B se cambia la MAC (la IP es igual ) a MAC fuente B MAC destino C

	- Estos dispositivos tienen una tabla de descriptores con enlaces directos que también asigna el administrador red que en este caso seremos nosotros

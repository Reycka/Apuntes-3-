	- Encaminamiento: por donde se mandan esos paquetes

- Datagrama:
	- Longitud total (16 bits)
		- Cabecera + datos
		- Siempre ocupa 16 bits
	- TTL:
		- Los paquetes tienen todos un TTL o tiempo de vida que indica el número total de encaminadores tienen como máximo para llegar a su destino, en caso de que este llegue a 0 el paquete se dejará de enviar y se considerará como paquete perdido teniendo que volver a enviarse
	- Fragmentación: 
		- Identificador: número de 16 bits que identifica el datagrama original
		- Flags:
			- MF More fragments si está a 1 indica que no es el último fragmenti
			- DF Don't Fragment si es 1 prohibe la fragmentación
		- Offset:
			- Posición de los datos del fragmento en los datos del datagrama original
			- En bloques es de 8 bytes
	- Direcciones IP origen y destino

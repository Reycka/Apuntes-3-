- Necesidad de adaptar el tamaño de un datagrama IP a la MTU (maximum transfer unit) del camino
- Opciones:
	- Fragmentación en el origen
	- Fragmentación en el camino (routers/encaminadores)
- El datagrama se reensambla en el destino
- La pérdida de un fragmento supone la retransmisión completa del datagrama
	- Fragmentación en el camino:
		- El encaminador divide el datagrama en fragmentos para ajustar su tamaño a la MTU de la red
		- Para cada fragmento se fija:
			- Las flags DF y MF
			- El offset de los datos

	- Fragmentación en el origen:
		- El host manda un datagrama con el DF a 1 y lo envía teniendo este que descubrir el mínimo MTU del camino, en casi de necesitarse una fragmentación

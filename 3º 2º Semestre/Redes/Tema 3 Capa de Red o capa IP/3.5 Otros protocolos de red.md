- ICMP Internet Control Message Protocol 
	- Protocolo para el intercambio de mensajes de control de la red
	- Hay 2 tipos de mensajes:
		- Mensajes de error
			- Informan de situaciones de error en la red (destino inalcanzable, tiempo excedido, problema de parámetro...)
		- Mensajes informatiovs
			- Permite intercambiar información sobre la presencia o el estado de un sistema (mensaje eco, anuncio o solicitud de router, redirecciones...)
	- Si que tienen cabecera IP ![[Pasted image 20260226093821.png]]
	- El tipo indica el TIPO del mensaje
	- El código Información adicional sobre el contenido del mensaje
	- El Checksum detecta errores en el mensaje
	- TIpos de mensaje: ![[Pasted image 20260226094049.png]]
	- La orden **ping*** envía el echo request y espera la recepcion del mensaje ICMP del Echo Reply
	
- SubRedes
	- Ventajas
		- Permiten aislar el tráfico
		- Protege el acceso a las distintas subredes
		- La comunicación entre estas es por router
		- Se puede dividir la red en áreas/departamentos
		- Cada área tiene un subconjunto de IPS
		- Las IPS las puede asignar el propio departamento
	- Ejemplo 84-86
	- Hay 2 formas de comunicar subredes:
		- Por router como encaminador
		- Hacer que uno de los Hostos sea el propio encaminador
			- En este caso solo hay una interfaz eth0 pero se le pueden asignar varias IPS
	- Las subredes pueden tener longitudes variables, es decir, puedes tener 3 subredes de una misma red y a su vez subdividirlas como quiera.
	- Es como el meme del tío que se duplica xd

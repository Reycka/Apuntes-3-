- ¿Qué es un servidor web?
	- Es la máquina o un programa informático que:
		- Almacena páginas web
		- Acepta grandes peticiones web de clientes
		- Genera una respuesta que envía al cliente
	- Se usa el protocolo HTTP
- Servidor web vs Servidor de aplicaciones
	- Un servidor web está diseñado para gestionar y servir contenido estático
	- Un servidor de aplicaciones gestiona y sirve contenido dinámico
	- Ejemplo diapos 6 - 7
- Navegador:
	- Reciben el contenido de HTML, lo interpretan y lo muestran
- Protocolo HTTP:
	- El navegador hace un get si lo encuentra lo manda y muestra el HTML mientras que si no lo encuentra manda el error 404, hace también algún POST si es que la web es interactiva
	- Cookies: archivos que almacenan la información del usuario para registrar lo que están haciendo, esto se guarda en lado del cliente.
		- Permiten el inicio de sesión, sin ellas no habría tiendas ni nada online
		- IMPORTANTE: Problema en HTTP, toda la información de la cookie ESTÁN VISIBLES SIEMPRE mientras que en el HTTPS van cifradas y el problema es que viajan por internet en claro (EN LAS COOKIES PUEDEN ESTAR CONTRASEÑAS GORDAS), a su vez agregan un código MAC para garantizar la Integridad ya que esas cookies se pueden modifcar por tanto al cifrarlo con MAC incluida garantizamos Integridad y Autenticidad
	- Importante: igual que no permitir el HTTP para el HTTPS  es fundamental permitir un correcto funcionamiento del cierre de sesión por parte del servidor
	- Para evitar problemas de sesiones que no se cierran de forma correcta (logout) simplemente cerrarla con la X (lo cual no indica al servidor que la cookie ya no es válida) se debe de implementar alguna forma que permita hacer ese sign out (boton manual) esto marca al servidor como cerrada y borra el ID de sesión del usuario

		- Inyecta un script malicioso en un contexto de confianza
		- En el cliente está basado en DOM
		- En el servidor:
			- ![[Pasted image 20260310144450.png]]

	- Cross-Site Request Forgery (CSRF) (no entra como tal pero mola conocerlo)
		- Aprovecha la sesión de un usuario en un servidor (Tengo la sesión abierta con el banco y veo algo por fuera en otra página que tiene un enlace pincho llevandome a otra página maliciosa que aprovecha mi sesión para hacerse pasar por mi sin que me entere)

		- Prevención:
			- Validar el encabezado Referer
			- Atributo Samesite de las cookies
			- Uso y validación de tokens secretos (CSRF token)
- TLS
	- HTTPS = HTTP + TLS, se trata de un ciclo de cifrado simétrico que garantiza la autenticidad y la confidencialidad 
	- Protege a los clientes en cuanto Autenticidad, integridad y confidencialidad pero al servidor nanai de la china 
	- UN SERVIDOR SEGURO PROPORCIONA SERVICIOS SEGUROS A LOS CLIENTES
	- Ciclo:
		- 1º El cliente le pide al servidor su certificado digital
		- 2º Si el certificado es válido el servidor le pide al cliente que se registre
		- 3º Una vez hehco eso se genera una clave única por sesión (https)

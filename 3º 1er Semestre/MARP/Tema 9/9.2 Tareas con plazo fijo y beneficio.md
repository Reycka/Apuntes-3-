- Todas las tareas tardan lo mismo pero tienen limite de tiempo y beneficio distinto, el objetivo es maximizar el beneficio obtenido
- Características:
	- Hay n tareas
	- Cada una tiene un plazo pi y un beneficio bi
	- No tienen que realizarse todas las tareas siempre y no se pueden hacer tareas en simultáneo
	- El objetivo es maximizar el beneficio total obtenido con estas características
- Estrategia:
	- Un conjunto de tareas es factible si existe alguna secunecia de ejecución admisible, que permita realizar todas las tareas dentro de sus plazos
	- Ordenamos las tareas de mayor a menor beneficio
	- Vamos aceptando las tareas en este orden que cumplan que el conjunto de la secuencia sea factible (no hay 2 tareas que se deban hacer el mismo día)
- Tests de Factibilidad
	- Test 1: Las tareas que se escogen se van rellenando en el vector en orden (1,2,3) según se puedan colocar, si solo tengo el hueco 2 libre pero la tarea es de huceo 1 entoncés no es factible
	- Test 2: Colocar las tareas y colocándola siempre el último día posible que se puede colocar siempre que este no este pillada, si lo está se coloca en el anterior hasta encontrar el primer hueco libre
		- Tenemos un vector libre que indica los días libres de cada una de las tareas de forma que inicialmente empiece vacía donde libre[i] = i
		- Un vector plan donde guardamos las tareas
		- Tenemos también un ConjuntoDisjunto de partición donde se busca el día libre entre el mínimo de N, y tareas[i].plazo, esto genera un ínidce que se pregunta a libre, si el resultado es 0 signifca que la tarea no entra
		- Complejidad N(lg*N)

- Definición y usos:
	- Método algorítmico por el cual se buscan todas las posibles soluciones a un problema almacenando en una tabla de valores dichas soluciones, permitiendo optimizar la consulta de dichos valores para finalmente obtener la mejor solución
	- Se utiliza siempre que no existe estrategia voraz para algún problema
	- Un ejemplo es el cálculo de números combinatorios
- Planteamiento:
	- Hay un conjunto de elementos y queremos saber la cantidad de combinaciones de estos, tenemos 2 opciones con esto:
		- Tomamos el elemento i
			- Nos quedan r-1 elementos que son exactamente (n-1 sobre r-1) para el ejemplo de números combinatorios
		- No tomamos el elemento i
			- Nos quedan r elementos todavía ya que no hemos puesto ninguno sin incluir el elemento i (n-1 sobre r)
	- Con esto tenemos siempre que el total viene dado por la suma de ambas partes para todos los valores de I 
	- No obstante sin la tabla estamos resolviendo nuevamente problemas que ya han sido solucionados, por lo que la forma ideal de resolver este problema sería usando una tabla que compruebe que ese caso ya haya sido solucionado
- Características:
	- Se usa una tabla para almacenar los resultados de subproblemas ya resueltos
	- La tabla tiene tantas dimensiones como argumentos tenga la recurrencia
	- El tamaño de cada dimensión coincide con los valores que puede tomar el argumento correspondiente
	- Cada subproblema se asocia a una posición de la tabla

- Modalidad:
	- Descendente
		- Mantiene el enfoque recursivo
		- La función recibe como parámetro de entrada/salida, la tabla donde se almacenan las soluciones a subproblemas ya resueltos
		- Antes de resolver de manera recursiva un subproblema, se mira en la tabla por si ya se hubiera resuelto
		- Tras resolver un subproblema recursivo, su solución se almacena en la tabla
		- Necesidad de saber si un subproblema está resuelto o no
		- Resolución:
			- Se toman los casos base:
				- Los del enunciado y se hace return del valor
				- Si el valor de la tabla ya fue visitada (!= -1)
			- Se toman los casos recursivos:
				- Dependen del enunciado
			- La matriz tiene tamaño N +1 y M + 1
			- Complejidad de (N * M) Donde N numero de filas y R número de columnas
	- Ascendente:
		- Cambiar el orden en el que se resuelve los subproblemas
		- Se comienza por resolver todos los subproblemas más pequeños que se puedan necesitar, para ir combinándolos hasta llegar a resolver el problema original
		- Los subproblemas se van resolviendo recorriéndolos de menor a mayor tamaño
		- Todos los posibles subproblemas de tamaño menor tienen que ser resueltos antes que los de tamaño mayor
		- Muy útil para disminuir el tamaño de la tabla
		- Resolución:
			- Se realizan 2 bucles for en simultáneo de tal manera que antes del segundo for se resuelve el subproblema pequeño para que el resto se puedan realizar (casos base) y luego se resuelven en el siguiente bucle for todos los subproblemas que se puedan rellenar con el caso base de i resuelto
			- Finalemnte la resolución se encuentra en la posición N M
			- Compljidad O(N * M)
			- Con este método podemos convertir la matriz en un vector que almacena únicamente las soluciones necesarias y las vaya actualizando hasta llegar a la solución final
				- De esa forma la primera parte de la fila del vector serán las posiciones necesarias para comparar y al final estarán los resultados de la nueva fila
				- Para esta forma recorremos de izquierda a derecha
				- Ahora la complejidad espacial ha bajado a O(M)

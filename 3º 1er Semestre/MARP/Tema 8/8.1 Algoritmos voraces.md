- Definición y usos:
	- El objetivo de estos algoritmos es el de minimizar el tiempo de ejecución de un programa para encontrar la solución más óptima a un problema aplicando una serie de reglas predefinidas que permiten obviar algunas soluciones las cuales sabemos antes de ejecutar cualquier cosa que van a ser inválidas
	- En cada paso que se de en el algoritmo se elije siempre el mejor paso en cada parte
	- Se pueden resolver problemas como el del cambio de monedas, o encontrar el ARM (kruskal) caminos mínimos (dijkstra), 
	- Método voraz:
		- Se trata de un método que construye una solución a través de una secuencia de pasos, hasta que se alcanza una solución completa al problema
		- En cada paso la elección debe ser:
			- Factible, es decir, tiene que satisfacer las restricciones del problema
			- Óptima localmente, es decir, la mejor opción local entre todas las disponibles
			- Irrevocable: es decir, no se puede cambiar en los pasos posteriores al algoritmo
		- Características:
			- Mientras se construye la solución se dispone de un conjunto de candidatos (de forma mental, no directa) generándose 2 conjuntos principales, los seleccionados (forman parte de la solución) y los rechazados definitivamente
			- Existe una función de selección (estrategia voraz) que indica cual es el candidato más prometedor de entre los aún no considerados
			- Existe un test de factibilidad que comprueba si un candidato es compatible con la solución parcial construida hasta el momento
			- Existe un test de solución que determina si una solución parcial forma ya una solución completa
			- Se tiene que obtener una solución óptima según una función objetivo que asocia un valor a cada solución
- Método de reducción de diferencias:
	- Se compara una solución óptima con la solución obtenida en el algoritmo voraz. Si ambas soluciones no son iguales se va transformando la óptima de partida de forma que continúe siendo óptima pero más similar a la del voraz.
	- Solución óptima: La mejor solución que podemos del problema
	- Solución voraz: Solución dada por el algoritmo voraz
	- Este método de reducción de diferencias es la forma de demostrar la optimalidad del algoritmo voraz
- Contraejemplos:
	- Son ejemplos que prueben que nuestra estrategia voraz es incorrecta, demostrando que la solución óptima de este ejemplo no coincide con la solución voraz.
- Resolución:
	- Buscamos como ordenar el vector de datos
	- Realizamos las medidas según el ejercicio resuelto
	- Comprobamos solución óptima con la voraz con algún contraejemplo

- Son colas de prioridad pero que permiten cambiar la prioridad de los elementos con un coste O(logN) sin tener que hacer pop y push de la cola completamente, implementadas con montículos
- Para ello la clase nueva IndexPQ utiliza índices para distinguir un elemento y otro, de forma que podemos saber cual es con dicho índice
- Operaciones:
	- Constructora IndexPQ(int TotalElementos)
	- push(int elemento, Prioridad) Añade a la lista un nuevo elemento O(log N)
	- update(int elemento, prioridad) Cambia la prioridad del elemento O(log N)
	- top() Devuelve el índice y la prioridad del elemento más prioritario O()1
	- pop() elimina el elemento más prioritario O(log N)
	- empty() comprueba si el vector está vacío O(1)
	- size() Total de elementos en la cola O(1)
	- Flotar y hundir iguales para montículos O(log N)
- Se almacenan en un vector igual que las colas de prioridad, con la diferencia que este vector es ahora de pares (índice,prioridad)
- Del mismo modo y para saber en que posición del vector está el elemento del índice hay un vector auxiliar de posiciones que indica donde están dichos elementos

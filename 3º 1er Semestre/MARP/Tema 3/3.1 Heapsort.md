- Se trata de un método de ordenación basada en un montículo.
- Inicialmente este algoritmo usa un vector y una cola de prioridad distintas para la ordenación de los mismos, no obstante eso duplica el coste en espacio por lo que para ahorrar esto, podemos usar el mismo vector como montículo auxiliar, provocando una mejora del espacio dejando la complejidad temporal (NlogN) idéntica pero con una complejidad espacial de O(1)!!!
- Para ello el Vector se transforma en un montículo y los elementos más prioritarios se van guardando AL FINAL del vector provocando que así un único vector almacene todo

- De Vector a Montículo:
	- Se van comparando los padres de las hojas y se van intercambiando con las operaciones de flotar con el hijo mayor (o menor depende del tipo de montículo) hasta generar el montículo, la complejidad temporal es de O(N) ya que el trabajo por los niveles actúa como sumatorio provocando que el coste que se puede pensar en un principio NlogN se convierta en N directamente donde N es el número de Nodos.
	- Se puede hacer usando la operacion flotar_max si se recorre de izquierda a derecha o la operacion hundir_max si se recorre de derecha a izquierda, cualquier otra solucion no garantiza montículo
- Ordenación del vector a montículo.
	-  Se empieza siempre por el primer elemento, haciendo pop del mismo, como se explica en el 2.2. Tras la ordenación con el hundir se sigue generando el montículo, mientras esto ocurre los últimos elementos del vector guardan ese elemento prioritario!!
	- El coste de este algoritmo si que es NlogN ya que hundir tiene coste logN y se hace N veces (1 vez por cada Nodo) en el peor de los casos

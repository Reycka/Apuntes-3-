- Definición y usos:
	- Dado un Digrafo valorado, calcular el camino de coste mínimo entre cada par de vértices.
	- Si los pesos son positivos y el grafo es disperso (pocas aristas), usando dijkstra obtenemos un algoritmo de coste O(V A log V)
	- No obstante en el resto de casos no podemos usar este método ya que si es denso el coste asciende a V^3 A log V por lo que en su lugar usamos el algoritmo de floyd
	- Este algoritmo se puede utilizar para encontrar ciclos negativos, ya que en la diagonal principal se encontraría un número negativo
- Algoritmo de Floyd:
	- Resuelve este problema para todos los casos posibles con un coste O(V^3)
	- La diagonal principal solo tiene 0 (porque no hay autoaristas)
	- El valor en la posición i,j si hay arista es el coste 
	- Si no hay arista su valor será infinito
	- Este algoritmo utiliza un pequeño truco que es delimitar los vértices por los que puede pasar, de forma que se define esta recurrencia:
		- C^k (i,j) --> Coste mínimo para ir de i,j pudiendo usar todos los vértices de 1 a k
- Recursión:
	- Va en función de K:
		- El caso base es cuando no quedan vértices intermedios
			- 0 si i == j
			- Infinito si no hay arista i a j
		- Tomo el camino que pasa por K
		- No tomo el camino que pasa por K
		- Me quedo con el menor de estos
		- Es tan sencillo como comparar el coste de i,j con el coste de i,k + k,j
		- Suponemos que no hay ciclos

	- Cumple el principio de optimalidad 
- Tabla:
	- Se necesitan 3 dimensiones:
		- Para la K, la I y la J
		- En ascendente rellenamos la posición K = 0
		- Para contestar a la pregunta i,j necesitamos estos datos (i k,i j, k j) de la matriz anterior (la casilla de la fila superior), que aquí es una matriz N* N que tiene submatrices dentro
		- La K va creciendo de 1 a V y se puede rellenar en cualquier orden
		- Se puede mejorar el espacio
		
	- Si mejoro el espacio necesito 2 dimensiones:
		- Tenemos una casilla que es una matriz que representa C ^k-1 donde para rellenar la posición i,j de K tengo los datos i,j i,k ,j k de esta matriz
		- No optimiza nada pero simplifica el código
		- En este formato podemos observar como los valores de las filas de ambas matrices coinciden por lo que podemos observar una cosa:
			- El valor en C^k k,j = min(C^k-1 K,j, C^k-1 k k + C^k-1 (k,j)) que es el mismo valor que el que se encuentra en C^k-1 k,j
			- Esto es así porque el valor de C^k-1 de kk está en la diagonal principal y por ende vale 0 y nos queda el mínimo entre 2 valores idénticos que es C^k-1 k,j y esto de forma recursiva
		- Esto defiende el que se pueda recorrer en cualquier orden
- Reconstrucción:
	- Se puede hacer con las optimizaciones de espacio
	- Para ello vamos a utilizar la estrategia del vértice anterior de grafos
	- Para definir el anterior de j usamos lo siguiente:
		- A^K (i,j) = vértice anterior de j en el mínimo que puede usar cualquier vértice que vaya de 1 a k
		- Existe camino siempre que i != j y el valor de la tabla sea distinto de infinito en caso de que K = 0
		- Si K != 0 hay que hacer la distinción de casos en caso de si me conviene usar K o no, si me conviene el anterior es K y si no el anterior es i
	- Lo bueno de esto es que A se rellena a la vez que C entoncés podemos realizar la reconstrucción después con un coste de O(V) ya que el camino sigue hasta llegar de i a j
- Complejidad:
	- Coste espacio O(1)
	- Coste tiempo O(V^2) + O(V^3)

- Definición y usos:
	- Hay n objetos, cada uno con un peso (entero) pi > 0 y un valor real vi > 0
	- La mochila soporta un peso total (entero) máximo M > 0
	- Y el problema consiste en maximizar el valor de los objetos de la mochila
	- No obstante aquí no puedo partir los pesos, por lo que siempre tomaremos 2 casos recursivos para este ejercicio, tomo el objeto, no tomo el objeto
	- En las soluciones no importa el orden en el que los objetos son introducidos
	- Las soluciones dependen de los objetos que tengamos disponibles para introducir en la mochila y del peso que soporte esta.
	- De esta forma la tabla almacenará el precio máximo que se puede conseguir con ese peso en concreto
	- Cumple el principio de la optimalidad de Bellman, por lo que se puede plantear de forma recursiva
	- Se puede realizar con un algoritmo descendente, ya que al no poder optimizar espacio no es interesante hacerlo de forma ascendnete 
- Recursión:
	- Caso base 1: No queda peso disponible en la mochila
	- Caso base 2: No quedan objetos 
	- Caso recursivo 1: Tomo el objeto y lo meto en la mochila
	- Caso recursivo 2: No tomo el objeto y no lo meto en la mochila

	- Muy similar al de las monedas, si tomamos el valor de mochila(i-1,j),mochila(i-1,j-pi) + vi, si es el primer valor el máx significa que no tomamos ese objeto, pero si es el otro entonces si que lo tomamos, no obstante aquí no hay truco para la reconstrucción con solo un vector como si pasaba con las monedas

	- Mochila O(N*M)
	- Reconstrucción O(N)

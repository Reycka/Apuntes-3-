- Recorrido en profundidad
	- Es un recorrido del grafo, es decir, partiendo de un punto A se van recorriendo los distintos vértices que están unidos a ese vértice hasta que se han visitado todos generando así un camino
	- Es conocido como DFS por sus siglas en inglés
	- Imita la resolución de un laberinto de forma recursiva
	- Se usa también para la comprobación de propiedades de grafos así como para identificar un tipo u otro
	- Estructura y resolución:
		- Tenemos 2 vectores, 1 para guardar el vértice anterior, y otro para guardar un booleano que me indica si el vértice ya había sido visitado anteriormente
		- La forma de realizar el recorrido es recorrerse todos los vértices adyacentes al vértice original de forma que si A se conecta con B y C y B se conecta con C el recorrido empezaría con el B y después desde B comprobaría A que ya está visitado y luego con C, después se va a C y como no quedan más adyacentes sin visitar se vuelve a B que le pasa lo mismo y luego a A que le vuelve a pasar lo mismo!!!
		- ![[Pasted image 20260605161403.png]]
		- Como se observa, este grafo no es Conexo, ya que desde el 0 no se pueden visitar todos vértices
		- Para reconstruir el camino se pregunta el anterior desde el vértice del que se quiere saber el camino (el final) guardando primero ese vértices en la última posición del vector y después apuntamos el anterior al mismo, el anterior del anterior... UTILIZAMOS UNA COLA DOBLE O DEQUEUE
		- La complejidad de este algoritmo es de O(V + A) siendo V el número de vértices y A el número de aristas
- Máxima componente conexa

	- Para ello debemos tener almacenado un valor que indica la cantidad de amigos que hay en ese recorrido, de  forma que nos quedaremos siempre con el de mayor valor de cada 1

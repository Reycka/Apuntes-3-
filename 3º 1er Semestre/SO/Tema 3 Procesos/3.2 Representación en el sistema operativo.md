- Ciclo de vida:
	-  ![[Pasted image 20251017115404.png]]
	- 1º El proceso se crea
	- 2º El proceso se ejecuta o está listo para ejecutarse (running)
	- 3º Si paras el proceso te vas al Task_Stoped sacandolo de la lista de ejecutables hasta que lo reanudes
	- 4º Si se Interrumpe el proceso este se puede reanudar con la diferencia de que este se puede interrumpir si el usuario lo desea (Cualquier cosa de E/S)
	- 5º Si se Interrumpe el proceso este se puede reanudar con la diferencia de que este NO se puede interrumpir (Si por ejemplo esta buscando algo inexistente y está intentando devolverlo)
	- 6º Puede finalizar el proceso, conocido como Zombie (ya que sigue presente pero no tiene ningun tipo de uso real (No realizan funciones vitales))
	- 7º Una vez finalizado lo elimina de la lista de procesos
	
- Bloque de control de proceso (PCB)
	-  El bloque de control de proceso (PCB) es una estructura de datos que contiene toda la información necesaria para gestionar el proceso
	- En linux se tiene un PCB con:
		- Identificadores del proceso y propiettario
		- Estado del procesador
		- Planificación
		- Memoria virtual
		- Entrada/Salida
		- Contabilidad
		
- Identificadores:
	- **Process ID(PID)** Identificador único que se asigna en la creación. Se utiliza para identificar al proceso en las llamadas al sistema
	- **Parent Process ID(PPID):** Identificador del proceso que lo creó
	- Identificadores de **Grupo de Procesos(GID) y Grupo de Sesisón (SID):** Son abstracciones que permiten a la shell implementar control de trabajos
	- El comando ps devuelve los procesos que están en el sistema
	- **Propietario:** Identificador del usuario (UID) y grupo (GID). Identifican al propietario del proceso.
	- Identificadores efectivos del usuario (EUID) y grupo (EGID). Estos identificadores son los que usa el SO para el control de acceso.

	- Contexto del sistema de ficheros 
		- **Directorio de trabajo:** El cual se saca a traves de un pwd que resuelve toda ruta relativa en el proceso
		- **Directorio raíz (root)** chroot(1,2) usado para interpretar rutas absolutas. Kimita el espacio de nombres del SF del proceso
		
	- Contexto de Ejecución: 
		- Registros de la CPU:
			- Contador de programa
			- Puntero de pila
			- Registros de propósito general
			- Registros específicos del procesador
			
	- Gestión de Memoria
		- El PCB tiene el perfil completo de memoria virtual del proceso
		- Las páginas de memoria incluyen los segmentos de código, datos, heap,pila,librerías...
		- A su vez se incluyen los argumentos del programa y e de entorno
		
	- Sistema de ficheros /proc
		-  /proc expone las estructuras del kernel en un árbol
		- La PCB se encuentra en un subdirectorio de /proc --> /proc/PID
		- El comandos ps(1) y top(1) muestran la información de esos procesos

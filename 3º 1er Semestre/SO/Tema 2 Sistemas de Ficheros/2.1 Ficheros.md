- 2.1.1 Conceptos de fichero:
	- Un fichero es una **abstracción** que permite almacenar y **manipular información** como una secuencia de bytes agrupados bajo un mismo **nombre** 
- 2.1.2 Atributos y tipos. Permisos
	- Dentro del SO el fichero se representa con la estructura i-nodo, que se encarga de gestionar los ficheros:
		- Las particiones son ficheros (2 partes Major(driver) y Minor(dispositivo)) 
		- Localiza donde se encuentra el fichero 
		- Tipo de fichero y modo
		- El identificador del usuario y el grupo
		- Número de enlaces al fichero
		- El tamaño total de bytes
		- El tamaño de bloques normalmente de 512
		- Marcas de tiempo para:
			- Creación
			- acceso al fichero
			- Modificación
			- Cambio
	- Diapo 6  
	- Permisos de ususario:
		- Atributo modo del i-nodo 
		- Permisos especiales en **ficheros ejecutables (+x)**
			- setuid: El UID del proceso se establece al UID del propietario del fichero CAMBIA X POR S 
			- setgid: El GID del proceso se establece al GID del propietario del fichero
			- Estos permisos provocan que el fichero se cree con la identidad del usuario y el proceso en lugar de los del padre
		- Permisos especiales en **directorios** 
			- Ejecución (x): permite cambiar el directorio (cd)
			- setgid: El grupo GID del fichero se crea con el GID del directorio (no el del proceso)
			- sticky: Los ficheros solo se pueden eliminar y renombrar por el propietario del fichero
		- Internamente se representa como un grupo de bytes, siendo r-x, -w-, rw- un ejemplo donde internamente se traduce como 5|2|6 ya que r-x es 101 en binario osea (5)8 donde r-x son los permisos del usuario, w es el grupo y rw son los permisos del grupo
			- r/w representa lectura
			- x representa escritura
			- - Este simbolo indica que no tienes permisos
		- EJ: Para colocar los permisos usamos chmod 526 prueba --> ESTO DEVUELVE LO DE ARRIBA, los permisos de directorio si que permiten eliminar el fichero, ya que modifico el directorio, no el fichero

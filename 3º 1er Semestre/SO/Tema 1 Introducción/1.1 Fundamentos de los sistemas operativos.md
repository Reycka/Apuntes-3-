- 1.1.1 Kernel
	- Programa que gestiona todo el sistema: dispositivo hardware, memoria, planificación de la CPU...
	- Modelo de kernel monolítico diapo 5
	
- 1.1.2 Llamadas al sistema
	- El kernel se ejecuta en el modo privilegiado o kernel de la CPU con acceso a todos los recursos del sistema
	- El resto de las aplicaciones se ejecutan en modo usuario con acceso restringido al sistema
	- Llamadas al sistema:
		- El mecanismo que permite ejecutar acciones privilegiadas
		- Inclusión de interfaces para acceder a la funcionalidad
		- Podemos usar write(2), read(2) y stat(2) entre otras
		- strace(1) permite ver las llamadas de un programa al sistema
		- man para acceder al manual, se puede ver parte concreta con (numero de pagina) write
		
- 1.1.3 Procesos
	- Aplicación que se está ejecutando en el sistema operativo
	- Características
		- Espacio de direcciones de memoria
		- Descriptores de ficheros en uso
		- Registros de la CPU
		- Pila de memoria
		- Estado de los atributos que usa el Kernel
	- Este desarrollo permite al Sistema Operativo:
		- Mejora la eficiencia de los recursos
		- Permite el uso del tiempo-compartido de los recursos para ejecutar varios programas a la vez
		- Permite el desarrollo de los sistemas a tiempo real, es decir, sabe cuanto va a tardar (recurso planificado del sistema)
		
- 1.1.4 Memoria Virtual
	- Abstracción que permite a los procesos y al kernel creer que tienen memoria infinita
	- Se implementa por la CPU y el sistema operativo
	- Permite:
		- La multitarea evitando así conflictos entre las aplicaciones
		- Sobresuscripción de la memoria principal usando almacenamiento secundario
		
- 1.1.5 Planificadores
	- Responsable de asignar los procesos a las CPUs del sistema
	- Objetivos
		- Dividir los ciclos de CPU entre procesos activos
		- Encargara de dar prioridad a las aplicaciones, esto es dinámico, es decir va cambiando
		
- 1.1.6 Sistemas de Ficheros
	- Permite organizar los datos en carpetas y ficheros
	- El sistema de ficheros ofrece una interfaz (POSIX)
	- En UNIX todo los sistemas de ficheros parten del mismo padre, mientras que en Windows se diferencian por letras
	- El kernel soporta distintos sistemas de ficheros
	
- 1.1.7 Controladores de Dispositivos
	- El kernel interactúa con los dispositivos conectados al sistema mediante controladores
	- Normalmente permiten carga dinámica
	- Tipos:
		- Modo caracter (raw): Acceso secuencial sin buffer de cualquier tamaño 
		- Modo bloque: Acceso aleatorio en bloques

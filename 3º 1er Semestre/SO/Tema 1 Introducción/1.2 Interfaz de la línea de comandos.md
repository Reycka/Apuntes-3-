- 1.2.2 Variables de Entorno

> [!NOTE] IMPORTANTE
> La shell en la que estoy se representa con un .
> Para averiguar cual es mi path se usa $PWD
> La variable ? se utiliza para comprobar si la ejecución ha sido exitosa (0) o no (cualquier número)
> Para crear variables name="valor"
> bash hace que se pase de un proceso a otro, cada variable está ubicada en su proceso específico
> No obstante si quieres pasar una variable a otra puedes usar export name="valor"
> Los corchetes agrupan las cosas [] por lo que dentro de esos corchetes busca todo lo que tenga alguno de los caracteres que estén en dicho corchete, mientras que si no los hay busca la secuencia en concreto
> El @ me permite ver todos los elementos de algo
> +x es el permiso de ejecución
> Para obtener el código de retorno del programa se usa el $?, si la ejecución es exitosa se puede hacer un echo "TRUE" que equivale al $? si solo si la ejecución es exitosa




- Se usan para pasar información entre entornos y procesos

	- Ejemplos:
		- USER
		- HOME
		- SHELL
	- Se acceden a las variables con un dolar 
	- El valor de la variable se fija con el op = 
	- 1.2.2.1 Entrecomillado en la Shell
		- Comillas dobles "definen cadenas con resolución variables"
				"El usuario es $USER"
		- Comillas sencillas 'definen cadenas sin resolución de variables'
		        'El usuario es $USER' 
		-  Comillas de ejecución sustituye el valor por la ejecución del comando
				El usuario es `whoami` 
	- Los comandos son ficheros ejecutables
	- Se buscan los ficheros en PATH
	- Un comando se puede ejecutar siempre usando la ruta (relativa (usando el . delante )o absoluta (la ruta tal cual))
	- 1.2.2.2 Comandos Importantes

	- 1.2.3 Redirecciones y Tuberías
		- Los programas pensados en UNIX son algo que hace solo una cosa pero que la haga bien, es decir su salida es la entrada de otro ya que los programas se comunican a través de flujos

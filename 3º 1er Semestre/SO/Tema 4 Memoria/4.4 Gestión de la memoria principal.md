- Asignación de marcos de página:
	- En un sistema de memoria virtual paginada el sistema de asignación es responsable de buscar marcos de páginas libres para un nuevo segmento
	- Aquí se utiliza el Binary Buddy (listas de orden 2)
	- ![[Pasted image 20260104190601.png]]
- Políticas de reemplazamiento:
	- Si no quedan marcos libres, que marco es el que debe reemplazarse
	- Su objetivo es minimizar el número de fallos de página, eliminando aquellas que no se volverán a usar
	- Determinadas páginas de memoria no se pueden reemplazar (kernel del SO)
	- Destacamos 2 ámbitos:
		- Local --> Solo se puede reemplazar un marco asignado al proceso que causa fallo
		- Global --> Se puede reemplazar cualquier marco
	- Algoritmos:
		- FIFO: First in first out, se sustituye el más antiguo de todos
			- ![[Pasted image 20260104190900.png]]
		- RELOJ: Igual que el FIFO pero ahora hay una flag que indica que la página sigue en uso, si se intenta sustituir la página con la flag en uno, esa página pasa a ser la nueva más reciente
			- ![[Pasted image 20260104191021.png]]
			- ![[Pasted image 20260104191134.png]]
		- LRU: Less reference use, se sustituye la página que haya sido accedida hace más tiempo
			- ![[Pasted image 20260104191152.png]]
	- Page Buffering:
		- Las páginas no referenciadas no se eliminan, se marcan como reclamables o páginas inactivas
		- Si esa página se vuelve a referenciar pasa ala cola de páginas activas, evitando su eliminación
